useState() 훅을 활용하여, text라는 상태 변수를 하나 생성. 그리고 input필드의 값이 변경되면 새 값이 상태에 개신 될 수 있도록 작성할 예정.

이 MyForm 컴포넌트는 폼 데이터가 리엑트에 의해 처리되기 때문에 _제어 컴포넌트_ 라고 불립니다.

제어되지 않는 컴포넌트에서 폼 데이터는 DOM에 의해서만 처리될겁니다.

setText(event.target.value); 문을 통해 input field에서 값을 가져와서 상태에 저장하고, 제출 버튼을 눌렀을 때 입력한 값을 alert로 출력  
이하는 예시

```jsx
import { useState } from 'react';

function MyForm2() {
  const [text, setText] = useState('');

  const handleChange = (event) => {
    setText(event.target.value);
    console.log(text);
  };

  // input field에 입력한 것을 submit하면 날려보낼 수 있도록 하는 함수를 작성
  const handleSubmit = (event) => {
    alert(`'${text}' 라고 입력하셨습니다.`);
    event.preventDefault();
  };

  return (
    <form onSubmit={handleSubmit}>
      <input type='text' onChange={handleChange} />
      <br />
      <br />
      <input type='submit' value='클릭하세요 🙏' />
    </form>
  );
}

export default MyForm2;
```

이상에서 어제 내용과 헷갈릴 수 있는데, 상태가 업데이트 도니다고 해서 매번 리렌더링이 일어나는 것은 아니다. 상태의 변화를 감지하여 리렌더링을 일으키키 위한 Hook은 useEffect()였다.

# TypeScript

- TypeScript는 MS사에서 개발한, 타입 시스템을 제공하는 JS 상위 집합에 해당합니다. 최근에 수요가 늘어나는 중이다. 기준으로 가장 빠르게 성장하는 프로그래밍 언어로 선정되었다.
- 특징:

1. 변수, 함수, 클래스에 대한 _자료형_ 을 정의할 수 있다. 이를 통해 개발 프로세스 초기에 오류를 포착하는 것이 가능하다.
2. 앱의 확장성을 향상시킬 수 도 있고, 코드를 더 쉽게 유지 관리할 수 있게 해준다.
3. 코드 가독성이 향상되고 코드 자체 문서화가 더 쉽다.

## 공통 타입

- TS는 변수를 초기활 할 때 변수의 타입을 자동으로 정의하는데, 이를 _타입 추론_ 이라고 한다.

```ts
let message = '안녕하세요';
// String 자료형이라고 ts가 타입 추론
message = 1; // 자료형이 string인데 number가 들어가 오류 발생
```

message에 string 값을 대입했을 때, ts는 해당 변수에 자료형이 string이라고 판단하여 타입 추론을 한다. 그런데 다음 줄에서는 message 변수가 number 자료형을 대입하니까 오류가 발생

TS의 자료형은 JS와 동일

1. string : 문자열
2. number : 숫자
3. bool : boolean

그런데 TS가 매번 암시적으로 타입 추론을 하게끔 하는 것이 아니라, 명시적 타입 설정을 하는 것도 가능.

- 명시적 타입 설정 방법

```ts

let age: number;
let isAvtive: boolean;

// 변수의 자료형 확인
console.log(typeof email); // 결과값 : string
```

그런데, 변수 타입의 타입을 모르는 경우도 있을 수 있습니다. 예를 들어서 외부 소스로부터 값을 받아올때는 명확한 자료형을 모를 수도 있습니다.

- 선택적 체이닝 연산자(?.)를 이용하면 오류를 일으키지 않고 null이거나 undefined일 수 있는 객체 속성 및 메서드에 안전하게 접근이 가능

```ts
type Person = {
  name: string;
  email: string;
  address?: {
    street: string;
    city: string;
  };
};
// 객체 생성
const person: Person = {
  name: '김이',
  email: 'kin2@test.com',
};

console.log(person);
// 그러면 오류가 생기는 부분
// console.log(person.address.street);  -> 오류 발생
// 그러면 선택적 체이닝을 통해 오류가 발생 자체를 억제할 수 있습니다.
console.log(person.address?.street);
```

이상의 결과값으로는 undefined가 출력되고, 오류는 발생하지 않습니다.

그럼 이상까지의 상황에서 TS를 적용했을 때의 이점은 기본적으로 JS사에서는 _실행을 시키고 나서야 오류 유무를 확인_ 할 수 있는 반면, TS 상에서는 개발 단계에서(즉, 테스트/ 실전 데이터 입력 이전 단계에서) 예측 가능한 오류를 발생시키지 않을 수 있다.

- 연산자를 이용한 서도 다른 타입을 병렬로 명시하는 방법

```ts
type InputType = string | number;
//정의한 타입을 이용

```

```ts
type Fuel = 'disel' | 'gasoline' | 'electric';
type NoOfGears = 5 | 6 | 7;

type Car = {
  brand: string;
  fuel: Fuel;
  gears: NoOfGears;
};

const car1: Car = {

  gears: 5, // 마찬가지로 5,6,7이 아닌 다른 숫자가 들어가도 오류 발생

```

이상의 예시는 연산자를 이용하여 _서로 다른 타입을 처리하는 타입_ 인 유니언 타입(Union Type)을 만드는 것이 가능합니다.

첫 번쨰 예시의 경우에는 string이거나 number일 수 있는 InputType이라는 유니언 타입을 생성했고, 두 번째 예시는 특정 stringdata(disel, gasiline, electic)이거나 특정 number data(5,6,7)만을 가질 수 있는 유니언 타입인 Fuel / NoOfgears를 생성했습니다.

### 이벤트

onChange라는 이벤트 핸들러 관련한 학습을 어제 했습니다. TS를 이용할 때는 이벤트 타입을 세분화할 필요가 있습니다.

```tsx
<input
  type='text'
  onChange={handleChange}
  value={name}
>
```

과 같이 정의되어있을 때, handleChange의 예시는 이하와 같습니다.

```tsx
const handleChange = (event) => {
  setName(event.target.value);
};
```

근데 이게 좀 난해한게, handleSubmit 때도 매개변수를 event라고 정해 줬습니다.

event가 특정한 HEML element와 관련이 있다는 점을 명시할 때 사용한느 명령어가 있는데,

```tsx
const handleChange = (event: React.ChangeEvent<HTMLINputElement>) => {
  setName(event.target.value);
};
```

그렇다면 form과 관련있는 onSubmit 이벤트 핸들러의 경우에는 어떻게 되냐면

```tsx
const handleSubmit = (event: React.FormEvent<HTMLFormElement>) => {
  event.preventDefault();
  alert(`Hello ${name}`);
};
```

과 같은 방식으로 작성합니다. 즉 저희가 임의적으로 e 혹은 event로 매개변수 명을 짓기는 하지만 어느 HTML 요소와 관련있는지에 따라 딸려있는 속서과 메서드가 서로 다르기 때문에 TS상에서는 이를 통제한느 편입니다.
